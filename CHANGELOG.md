# Changelog

## Unreleased

### Changed
- 扩展 `/删除文章` 支持批量参数：`/删除文章 <文章ID[,文章ID]>`，支持中英文逗号分隔，按“逐个处理并汇总”返回结果。
- 删除流程新增关联目录清理：除文章缓存外，还会尝试清理 `article_file_path`、抓取/发布 run_dir 与该文章全部任务 run_dir。

### Security
- 目录清理增加严格路径安全校验：仅允许删除位于 `article_cache_root`、`work_root`、插件数据根目录白名单内的路径；包含符号链接或越界路径会被拒绝并记录告警，防止危险路径删除。

## v0.0.27 - 2026-04-01

### Fixed
- 修复 `/发布文章` 在文章已解析成功但原抓取工作空间丢失时直接失败的问题；插件会优先基于数据库中的文章内容重建工作空间 `article.md`，再继续发布。
- 修复发布恢复误用抓取 `session/run_dir` 的问题；发布态上下文改为单独持久化，不再把抓取 `last_session_id/last_run_dir` 当作旧发布会话或发布工作空间。
- 当存在历史“发布失败” `session` 时，`/发布文章` 会先尝试继续旧发布会话；仅在明确识别为旧会话不可继续时，才会自动降级为一次新的发布执行，避免重复发文。
- 恢复发布工作空间时一律重建到新的 publish run dir，避免旧目录日志残留干扰本次分享链接识别。

## v0.0.25 - 2026-04-01

### Fixed
- 修复“继续获取文章”在 Codex 正常退出但尚未落出 `article.md` 时直接报错停止的问题；此场景下插件会基于同一 `session` 自动再继续 1 次，再决定是否停止任务。

## v0.0.24 - 2026-04-01

### Changed
- 默认消息识别表情从 `👍` 调整为 `👀`，继续沿用十进制 Unicode code point 的 `reaction_emoji_id` 配置写法。
- “文章获取成功”与“缓存命中回传”发送给群组的 Markdown 文件展示名改为 `${YYYY_MM_DD}_${文章一级标题}.md`；无一级标题时回退为 `${YYYY_MM_DD}_article.md`。

### Fixed
- 修复发送文件名额外截断到 80 字符的问题，改为仅清洗非法文件名字符，不再静默裁剪标题。

## v0.0.23 - 2026-03-30

### Added
- 新增“回复发布失败消息自动续跑”能力：回复含 `[AS-PUBLISH-FAILED]` 标签的失败消息并发送“继续 <补充指令>”，可自动 `resume` 同一 session 重试发布。
- 新增发布阶段图片失败自动重试：检测到图片上传失败时，会基于同一 session 自动重试最多 2 次再放弃。

### Changed
- 发布提示词增加强约束：图片上传首轮失败后至少再重试 2 次；发布成功最终回复必须包含知识库分享链接。

### Fixed
- 修复发布误报成功问题：不再把任意首个 URL 当作发布链接；仅识别“合理知识库链接”才判定成功。
- 当发布流程异常且未识别到知识库链接时，失败回执会附带 `task_complete` 原文，便于直接定位问题。
- 修复“发布失败回复可伪造续跑”风险：失败回执改为一次性 token，且仅允许原发起用户在回复机器人失败消息时继续。
- 修复空文本回复触发自动续跑的问题：仅显式回复“继续/resume”才会触发发布会话恢复。
- 修复机器人来源判定优先级问题：回复来源校验改为优先使用 sender/user_id，不再把 `self_id` 误当发送者身份。
- 修复结构化分享链接可信性判断回归：结构化 `share_url` 同样需要通过知识库链接判定（含域名白名单）才视为发布成功。
- 进一步收紧分享链接兜底判定：移除宽泛关键词，改为固定路由片段匹配；默认拒绝 `github/gitlab/gitee/bitbucket` 等公共代码托管域名，降低误报成功风险。
- 修复日志 URL 兜底误判窗口：发布结果不再扫描日志中的任意 URL；未配置白名单时仅在 `xws` 域名上执行路由兜底识别。
- 修复图片失败重试漏判：自动重试判断新增 `codex_error` 信号源，避免仅凭 `task_complete/诊断` 才能触发重试。
- 修复发布结果串读历史 rollout 风险：`task_complete` 提取改为绑定本次执行的 rollout 文件/会话与启动时间窗口，不再按 run_dir 盲选历史记录。

## v0.0.22 - 2026-03-30

### Added
- 新增“回复中断提示自动继续”流程：回复 `[AS-INTERRUPT]` 中断提示消息（或回复“继续”）可自动继续当前用户可恢复的任务；聚合提示中支持一次继续多个任务。

### Changed
- 抓取/继续任务在超时或意外中断时，用户回执统一为可解析的简短提示，不再回传详细错误文本。
- 新增群组级中断通知聚合：同一群短时间内多个任务同时中断时，仅发送一条聚合提示（含 `tasks` 标签字段）。

## v0.0.21 - 2026-03-30

### Fixed
- 修复“回复图片也触发总结”的误判：仅当被回复内容为纯文本 URL（去掉首尾空白后仅保留一个 `http/https` 链接）时才触发总结流程。
- `reply` 仅包含 `id` 的场景改为统一规则：`get_msg` 回查内容也必须满足“纯文本 URL”才触发。

## v0.0.20 - 2026-03-27

### Fixed
- 修复 `/每周总结` 链接来源错误：优先使用发布成功后的知识库分享链接（`publish_share_url`），不再默认回传原文地址。
- 发布成功后会持久化分享链接到数据库；周总结对历史记录会尝试从运行日志回溯分享链接并自动回填。
- 补齐周总结链路对分享链接缺失场景的兜底逻辑，降低“总结可用但链接非知识库链接”的回归风险。

## v0.0.19 - 2026-03-27

### Added
- 新增 `/每周总结` 命令：统计触发时刻向前 7 天内所有用户“已发布”文章，按领域输出“标题 + 访问链接”清单。
- 新增周总结两阶段 Codex 流程：先执行链接有效性校验，再执行主题归类与模板化输出。
- 仓储层新增近 7 天已发布文章查询能力，用于周总结候选集构建。

### Changed
- 周总结链路固定使用 `gpt-5.4` + `low`，并通过 `$post-article-to-xws-knowledgebase` 执行链接有效性判断（仅 `2xx` 视为有效）。
- 帮助命令与 README 文档同步新增 `/每周总结` 使用说明。

### Fixed
- 修复周总结链接校验结果解析的鲁棒性：支持 `valid_*` 与 `invalid_*` 结构化字段，并可由无效集合反推有效文章集合。

## v0.0.18 - 2026-03-27

### Added

### Changed
- front matter 识别增强：兼容 BOM、CRLF 与 EOF 结尾场景；抽取可读文本路径复用同一清理逻辑。

### Fixed
- 修复自动发布调用链参数透传错误：自动发布不再触发 `publish_article_command` 的关键字参数异常。
- 自动发布触发时补充识别表情回执，与“获取文章”流程保持一致。
- 自动发布出现异常时新增简短失败通知，避免仅日志可见。

## v0.0.17 - 2026-03-26

## v0.0.16 - 2026-03-26

- 修复 `/发布文章` 成功无回执问题：发布成功后会在当前会话返回成功通知，并包含文章 ID、发布目标与可解析到的分享链接。

# Changelog

## Unreleased

## v0.0.14 - 2026-03-24

### Fixed
- 修复发布分享链接在日志尾随反引号场景下清洗不完整的问题，避免链接多出 `` ` `` 字符。
- 修复发布流程消息策略：仅发送一条“开始发布”提示，不再发送发布进度/结束结果提示。

### Changed
- 解析成功（新抓取与缓存命中）后新增发布引导消息：按群聊/私聊展示发布命令，并附当前默认发布配置与设置命令。
- 发布引导消息发送后补充 `stop_event()`，降低其他插件误介入风险。
- 获取文章列表支持展示“完成但未发布成功”的记录（完成项最多 10 条）。
- 新增配置项 `prefix`（默认 `slfk`）用于群聊发布提示命令前缀。
- 移除已失效的发布进度配置项文档暴露，保持实现与配置契约一致。

## v0.0.13 - 2026-03-24

### Fixed
- 修复发布流程在 Codex 进程快速失败时仍可能先发送“文章总结中”进度消息的问题，避免与失败提示冲突。

### Changed
- 发布流程（`/发布文章`）固定采用“启动立即一次 + 每 60 秒”进度播报策略。
- `codex_progress_report_seconds` 配置语义明确为“仅抓取/继续流程使用”，并同步文档说明。

## v0.0.12 - 2026-03-24

### Added
- 新增发布相关命令：`/发布文章`、`/默认发布空间`、`/默认发布团队`、`/默认发布知识库`、`/默认发布`、`/文档总结帮助`、`/删除文章`。
- 新增用户级默认发布配置表 `user_publish_defaults`，支持按用户保存默认空间/团队/知识库。
- 新增发布提示词配置 `codex_publish_prompt_template`，用于驱动 `$post-article-to-xws-knowledgebase` 发布流程。

### Changed
- 文章抓取成功后自动进入“待发布”发布状态；发布失败记录失败状态，发布成功后回到“待发布”。
- `articles` 表新增创建者与发布状态字段，支持删除权限校验与发布状态记录。
- `/删除文章` 增加“仅创建者可删”校验，对历史无创建者数据回退为“最早任务用户”判定。

## v0.0.11 - 2026-03-23

### Added
- 为 `article_tasks` 增加并持久化 rollout 统计字段：工具调用次数、额外搜索次数、累计 token、进度播报次数。
- 新增任务统计落库方法 `update_task_rollout_stats`，用于总结过程中的实时指标记录。

### Changed
- 进度消息格式调整为面向用户的中文模板：展示已耗时分钟数、播报次数、工具调用、搜索次数、token 与原文地址。
- rollout 解析逻辑增强：结束前会 drain/flush pending 缓冲，降低尾部事件丢失概率。
- token 统计来源改为读取 rollout `event_msg.token_count.info.total_token_usage.total_tokens` 的最新值。

## v0.0.10 - 2026-03-23

### Fixed
- 将单文件 `main.py` 重构为 `main + handlers + service + utils` 分层，入口仅保留注册与路由装配。
- 调整插件内部导入策略：优先包内相对导入，并在无包上下文时回退到绝对导入，避免路径解析失败。
- 统一仓库内模块引用路径，修复 `repository/service/handler` 在不同加载上下文下的导入兼容性。
- 调整 Codex 默认非交互/续跑命令为 `--yolo` + 继承环境变量的容器友好权限，避免 `bun` 与本地监听在默认沙箱下触发 `EPERM`。

## v0.0.9 - 2026-03-22

### Fixed
- 新增 SQLite 持久化能力，支持文章缓存与任务状态落库。
- 新增 `/获取文章列表`、`/继续获取文章 <id>` 命令。
- 增加插件生命周期中断：卸载/停止时终止运行中的 Codex 进程并落库为停止状态。
- 增加同链接复用策略：已完成链接直接返回缓存结果，完成项不再出现在获取列表。
- 修复 rollout session 识别与绑定逻辑，避免错误会话写入导致恢复错位。
- 修复完成态任务重复创建问题，改为按用户+文章复用更新 completed 记录。
